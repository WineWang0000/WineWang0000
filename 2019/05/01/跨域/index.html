跨域

Posted on 2019-05-01

1.什么是同源策略？

浏览器出于安全的考虑，只允许与本域下的接口交互，不同源的客户端脚本在没有明确授权的情况下是不能读写对方的资源。那什么是本域呢？即同协议（如同是http、https），同域名（比如同是nihao.com），同端口（80端口）

2.什么是跨域？列举跨域有几种实现形式?

简单来说就是a网站的js代码想访问b网站，包括提交内容和获取内容，这就是跨域。但是出于安全问题，是被浏览器所默认禁止的。有俩种实现跨域的方式：1.JSONP：把要请求的接口放在script标签里，通过script标签加载数据的方式去获取数据当做js代码来执行。这个方式需要后端的配合。2.CORS：简单来说就是浏览器发现请求头不对就会给他加一个请求头”origin”，如果确定结束返回的数据就在其中加”Access-Control-Allow-Origin”。浏览器判断响应头是否含有origin的值。如果有我们就能拿到数据。

3.JSONP 的原理是什么？

JSONP是通过 script 标签加载数据的方式去获取数据当做 JS 代码来执行 提前在页面上声明一个函数，函数名通过接口传参的方式传给后台，后台解析到函数名后在原始数据上「包裹」这个函数名，发送给前端。换句话说，JSONP 需要对应接口的后端的配合才能实现。

4.CORS是什么？

CORS 全称是跨域资源共享（Cross-Origin Resource Sharing），是一种 ajax 跨域请求资源的方式，支持现代浏览器，IE支持10以上。 实现方式很简单，当你使用 XMLHttpRequest 发送请求时，浏览器发现该请求不符合同源策略，会给该请求加一个请求头：Origin，后台进行一系列处理，如果确定接受请求则在返回结果中加入一个响应头：Access-Control-Allow-Origin; 浏览器判断该相应头中是否包含 Origin 的值，如果有则浏览器会处理响应，我们就可以拿到响应数据，如果不包含浏览器直接驳回，这时我们无法拿到响应数据。所以 CORS 的表象是让你觉得它与同源的 ajax 请求没啥区别，代码完全一样。
